# ADR-0001: 購入ポイント付与システムの導入

| Field | Value |
|-------|-------|
| **ID** | ADR-0001 |
| **Stage** | Requirement |
| **Status** | Accepted |
| **Date** | 2026-03-22 |
| **Supersedes** | — |

---

## Context

新機能として、ユーザーが商品を購入した際に自動的にポイントを付与する仕組みを導入する。
顧客ロイヤルティの向上と購買促進を目的として、購入金額に応じたポイント還元と、
貯まったポイントを購入時の割引として利用できる機能が求められている。

ビジネス要件として以下のルールが確定している：
- ポイント付与率：購入金額の 1%
- ポイント利用最低条件：100 ポイント以上から利用可能
- ポイントの価値：1 ポイント = 1 円

## Decision

購入完了時に購入金額の 1% をポイントとして自動付与し、100 ポイント以上から 1 ポイント = 1 円として購入時に利用できるポイントリワードシステムを導入する。

具体的なビジネスルール：
- **付与ルール**：購入金額（税抜き・税込みは別途決定）の 1% を整数ポイントに切り捨てて付与する
- **利用条件**：保有ポイントが 100 ポイント以上の場合のみ利用可能
- **利用単位**：1 ポイント = 1 円として、購入金額から差し引く
- **自動付与**：購入完了（決済確定）のタイミングで自動的に付与する

## Verification (Harness)

```
npm run test:e2e -- --grep "point reward"
```

E2E テストで以下のシナリオを検証する：

1. **ポイント付与テスト**
   - 1,000 円の購入 → 10 ポイント付与されること
   - 10,000 円の購入 → 100 ポイント付与されること
   - 999 円の購入 → 9 ポイント付与されること（切り捨て）

2. **ポイント利用テスト**
   - 保有ポイント 100 以上 → 購入時にポイント利用オプションが表示されること
   - 保有ポイント 99 以下 → ポイント利用オプションが表示されないこと
   - 100 ポイント利用 → 購入金額から 100 円割引されること

3. **自動付与タイミングテスト**
   - 決済完了後にポイントが付与されること
   - 決済キャンセル時にポイントが付与されないこと

## Consequences

### Benefits（メリット）

- シンプルなルール設計（1%・100pt閾値・1pt=1円）により、ユーザーが理解しやすい
- 購買インセンティブが生まれ、リピート購入の促進が期待できる
- 自動付与により、手動オペレーションコストが発生しない

### Drawbacks（デメリット・トレードオフ）

- ポイント付与により実質的な商品原価率が上昇する（購入金額の最大 1% のコスト増）
- ポイント残高の管理・整合性保証のためにデータベース設計と排他制御が必要になる
- ポイント利用時の二重利用防止など、不正防止の仕組みが別途必要

### Side Effects（副作用・注意点）

- 税込み・税抜きどちらを基準にポイントを計算するかは別途決定が必要（ADR 追加推奨）
- ポイントの有効期限ポリシーが未定義のため、将来的に要件追加が必要になる可能性がある
- 購入取り消し・返品時のポイント回収ルールは別途定義が必要
