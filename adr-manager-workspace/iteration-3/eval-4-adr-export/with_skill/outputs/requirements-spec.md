## 機能要件

### ADR-0001: ロイヤルティポイント付与・利用ルール

**概要:** 購入金額の1%をポイントとして付与する。100ポイント以上から利用可能で、1ポイント=1円として決済時に充当できる。ポイントの有効期限は付与日から1年間とする。

**背景:**
購入促進施策として、ポイントプログラムを導入することになった。
ユーザーの購買継続率を高めるため、購入ごとにポイントを還元し、次回購入時に利用できる仕組みが必要。

**受入条件（Verification）:**
```
npm run test:e2e
```
E2E テストでポイント付与・利用・有効期限の全フローを検証する。

**備考（トレードオフ）:**

| 観点 | 内容 |
|------|------|
| メリット | 購買継続率の向上が期待できる |
| メリット | シンプルなルールで実装コストが低い |
| デメリット | ポイント原資のコスト管理が必要 |
| デメリット | 有効期限切れポイントの会計処理が発生する |
